# gdocs-sync

Translate a source **Google Doc** into any number of languages, automatically — with
a **stable document ID / share link** for each translated doc, so links you hand out
never break.

It reproduces the manual *copy → Tools → Translate document* routine (there's no API
for that button) using [Apps Script](https://developers.google.com/apps-script) +
`LanguageApp` — the same translation engine the menu uses. The script is **standalone**
(not bound to a doc) and driven by a **Google Sheet registry**, so it's fully reusable.

## How it works

1. The **registry sheet** is your control panel — one row per output doc:

   | builder | email | source_link | from | to_lang | mode | target_link | access | last_synced | status |
   |---------|-------|-------------|------|---------|------|-------------|--------|-------------|--------|
   | published | | …/master/edit | en | th | sync | …/edit | link | … | resynced |
   | Acme Co | a@b | …/master/edit | en | th | once | …/edit | link | … | created |

   - **mode = sync** → re-translated from the source on every run (a published shared translation).
   - **mode = once** → created once, then **never overwritten** — for a **builder's own copy**,
     which they read and write quotes into.
   - Leave **target_link** blank → a doc is created (copy of the source, translated) and its
     link written back (stable forever). `link` in **access** = anyone-with-link view.
2. `syncAll` processes every row: creates missing docs, re-translates `sync` rows, leaves
   `once` rows alone (so builder edits survive), and writes links back.
3. Targets are edited **in place** (by id) — ID, share link, permissions never change.

### Per-builder copies

Each builder gets their **own translated copy** of the master spec to read and quote into:

```sh
mise run gdoc:add "Acme Co" --lang th --email acme@example.com
```

This copies the master, translates it, creates a `mode=once` row, and returns the doc link.
The master can keep changing (its `published` translation re-syncs); builder copies are
frozen the moment they're made, so no one's quotes get wiped.

## Run it

- **Headless (full control):** `mise run gdoc:run` → triggers `syncAll` via a token-gated
  web app. No clicking.
- **From the editor:** open `gdoc:open`, pick `syncAll`, Run.
- **Nightly:** run `enableNightlySync` once → auto-syncs every night.

## Layout

```
src/
  appsscript.json   # manifest: V8, OAuth scopes, web-app + execution config
  config.js         # SETTINGS — registry sheet name, columns, seed row
  Code.js           # setupRegistry / syncAll / doGet (web trigger) / triggers
mise.toml           # gdoc:* + mcp:serve tasks (all secrets via fnox)
fnox.toml           # keychain contract (see below)
```

## Secrets (fnox)

Everything lives in the macOS keychain via [fnox](https://github.com/jdx/fnox); nothing
sensitive is committed. `.clasp.json` is gitignored and regenerated by `gdoc:link`.

| Item | What | Secret? |
| ---- | ---- | ------- |
| `GDOCS_SYNC_CLASP_RC` | `~/.clasprc.json` (clasp OAuth token) | yes |
| `GDOCS_SYNC_API_TOKEN` | token gating the web app | yes |
| `GDOCS_SYNC_SCRIPT_ID` | Apps Script Script ID | no |
| `GDOCS_SYNC_DEPLOY_ID` | web-app deployment id | no |
| `GDOCS_SYNC_WEBAPP_URL` | web-app trigger URL | no |
| `GDOCS_SYNC_REGISTRY_URL` | registry sheet URL | no |

## Setup (fresh machine)

```sh
mise run gdoc:login        # browser OAuth
mise run gdoc:auth-save    # stash credential in keychain
mise run gdoc:link         # write .clasp.json from the Script ID
mise run gdoc:push         # upload the code
# one-time, in the editor: run `setupApi`, copy the printed token, then:
fnox set -p keychain GDOCS_SYNC_API_TOKEN <token>
mise run gdoc:run          # verify headless trigger works
```

## Tasks

| Task | Does |
| ---- | ---- |
| `gdoc:add "<Builder>" [--lang th] [--email …]` | create a builder's own translated copy |
| `gdoc:run` | trigger syncAll headlessly (web app) |
| `gdoc:state` | read registry state (links + status) as JSON |
| `gdoc:setup-remote` | (re)initialise the registry sheet headlessly |
| `gdoc:push` / `gdoc:dev` / `gdoc:pull` / `gdoc:status` | clasp file sync |
| `gdoc:deploy` | update the web app to the latest pushed code (same URL) |
| `gdoc:open` | open the Apps Script editor |
| `gdoc:urls` | print live URLs |
| `gdoc:login` / `gdoc:auth-save` / `gdoc:auth-restore` / `gdoc:link` | auth/setup |
| `mcp:serve` | run clasp's MCP server (registered in `~/.claude.json` as `clasp`) |

## Notes / limits

- **Full regeneration:** each run rebuilds the target body and re-translates it. No
  partial sync (by design — targets aren't hand-edited).
- **Paragraph-level translation:** paragraph/heading styles preserved; a single bold
  word mid-sentence isn't. Better translation quality than word-by-word.
- Images, headers/footers, equations are left untouched.
- The web endpoint is public but inert without the secret token (kept only in the keychain).
